## 2025-03-05 - Missing API Authentication on Orchestration Endpoints
**Vulnerability:** The internal backend API (`api/server.py`) endpoints (`/extract` and `/check`) were exposed publicly without authentication. These endpoints orchestrate requests to paid third-party AI APIs (like OpenRouter or Gemini), creating a Denial of Service and API quota depletion risk where unauthorized requests could incur significant financial costs. Additionally, the error handlers exposed raw exception messages, which could leak internal configuration and stack traces to attackers.
**Learning:** Internal APIs designed for internal ecosystem clients (e.g., Telegram Bot) must implement basic layer-7 security (like API keys) if they access rate-limited or paid external resources. Exposing endpoints without authentication allows unconstrained external requests.
**Prevention:** Implement `APIKeyHeader` authentication leveraging secure constant-time comparisons (`secrets.compare_digest`) on endpoints orchestrating rate-limited external calls. Furthermore, ensure exception handlers sanitize errors via generic responses (e.g., HTTP 500 "Internal Server Error") while internally recording exceptions using robust logging mechanisms.
