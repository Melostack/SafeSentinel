## 2026-03-08 - [Missing Authentication on API Endpoints]
**Vulnerability:** The `/extract` and `/check` endpoints in `api/server.py` were missing authentication entirely, allowing anyone to bypass intent extraction and security checks while posing an immense risk for DoS and data extraction given the unbounded string requests.
**Learning:** External API interfaces for SafeSentinel were written without validating the caller. An attacker could flood the endpoints and run our systems into the ground or steal sensitive path computations without passing via the Telegram bot that would normally provide some form of obfuscation or rate limiting. Unbounded Pydantic models also compounded the DoS risk.
**Prevention:** Always enforce APIKeyHeader on sensitive endpoints in FastAPI and use `secrets.compare_digest` to perform constant-time comparisons. Ensure Pydantic fields have `max_length` defined. Catch-all exception blocks should never expose raw exceptions `str(e)` and must use internal logging (`logger.error`).
