## 2024-05-30 - Missing Authentication on Internal APIs
**Vulnerability:** The API endpoints `/check` and `/extract` in `api/server.py` were fully exposed without authentication, allowing anyone to trigger internal logic, potentially leading to DoS or unauthorized API quota depletion (e.g., OpenRouter API in `Humanizer`). In addition, the `CheckRequest` and `IntentRequest` models had unbounded string lengths.
**Learning:** Internal APIs that orchestrate rate-limited or paid external services must be authenticated by default to prevent unauthorized access. Relying on network-level obscurity is insufficient.
**Prevention:** Implement an API Key validation layer (`APIKeyHeader`) on sensitive endpoints. Set an environment variable like `SAFE_SENTINEL_API_KEY`, validate incoming headers using `secrets.compare_digest` to prevent timing attacks, and default to HTTP 500 if the key is unset. Also, always bound `Pydantic` models with `Field(..., max_length=N)` to prevent DoS via unbounded strings.
