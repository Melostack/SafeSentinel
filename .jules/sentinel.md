## 2024-05-20 - Missing API Authentication and Info Leakage in API Endpoints
**Vulnerability:** The internal FastAPI server endpoints (`/check` and `/extract`), which orchestrated requests to paid external LLM APIs, lacked authentication. Additionally, error handlers were returning raw stringified exceptions (`detail=str(e)`), which could leak stack traces or internal state to unauthorized actors. The endpoints also used unbounded string inputs on Pydantic models.
**Learning:** Internal APIs that consume quotas on third-party services must be authenticated by default to prevent API quota depletion and unauthenticated abuse (DoS). Similarly, raw error strings should never be exposed; they must be caught, logged securely, and masked behind generic user-facing messages. Always limit string input sizes on models to defend against DoS attacks.
**Prevention:** Implement `APIKeyHeader` authentication on all internal APIs. Catch specific or generic exceptions and mask them in HTTP responses. Use `Field(..., max_length=N)` in all Pydantic string inputs. Validate implementation correctness using `unittest` and HTTP request mocking.
