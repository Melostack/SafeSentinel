## 2026-03-09 - [Internal API Authentication Missing]
**Vulnerability:** The internal backend API (`api/server.py`) exposed sensitive analysis endpoints (`/check`, `/extract`) without authentication. While these endpoints might be intended for the internal Telegram bot, anyone able to access the API URL could submit unbounded requests, bypassing the bot and performing unchecked operations, potentially leading to DoS or scraping of internal intelligence. Furthermore, generic unhandled exceptions leaked stack traces/details through `detail=str(e)`.
**Learning:** Even internally-facing backend services separated by microservice architecture (Bot -> FastAPI) must have authentication configured to verify trust, especially in serverless or containerized setups where ports may be exposed. Additionally, exception handlers in API endpoints must never return raw error strings (e.g., `detail=str(e)`).
**Prevention:** Always implement service-to-service authentication (e.g., using `X-API-Key` headers matching a shared secret with `secrets.compare_digest`). Bound incoming Pydantic fields using `Field(..., max_length=N)` to prevent DoS. Use generic non-sensitive error responses alongside internal `logging.error(..., exc_info=True)`.
