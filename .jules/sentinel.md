## 2024-03-13 - [Information Leakage and Unbounded Inputs in API]
**Vulnerability:** The API endpoints were exposed to unbounded data inputs in Pydantic models (`CheckRequest`, `IntentRequest`) which could lead to DoS attacks. In addition, the global exception handler for the `/check` endpoint returned raw exception strings (`detail=str(e)`), which could leak sensitive stack traces or internal implementation details to the client.
**Learning:** Returning `str(e)` directly in error responses bypasses safe error handling and inadvertently exposes internals. Similarly, not explicitly defining limits for string inputs via Pydantic `Field(max_length=...)` allows processing of arbitrarily large inputs.
**Prevention:** Always define a specific `max_length` using Pydantic's `Field` for string inputs. When handling exceptions in API endpoints, ensure a generic, non-sensitive message is returned to the client while the actual error and traceback are logged securely internally using `logger.error(..., exc_info=True)`.
