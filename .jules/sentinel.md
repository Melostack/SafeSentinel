## 2024-05-24 - [Add Authentication to Sensitive Endpoints]
**Vulnerability:** The API endpoints (`/extract` and `/check`) in `api/server.py` lacked authentication, allowing any external party to call these sensitive functions without permission, leading to potential abuse or information leakage.
**Learning:** Sensitive operations must always have an authentication mechanism in place. It's critical to ensure a "secure failure state" when dealing with API key configuration (e.g. failing with HTTP 500 when `SAFE_SENTINEL_API_KEY` is not set). Additionally, using `secrets.compare_digest` prevents timing attacks on API key validation. Furthermore, error messages must not leak internal logic (`detail=str(e)` vs `detail="Internal Server Error"`).
**Prevention:** Always implement an authentication layer such as `APIKeyHeader` on sensitive routes. Log internal errors securely with `logging.getLogger(__name__).error(..., exc_info=True)` and return a generic error message to the client. Validate credentials safely using constant-time comparison methods. Ensure failure states act safely by denying access when expected configuration is missing.
