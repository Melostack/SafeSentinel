## 2024-05-18 - [Missing Authentication on Resource Intensive APIs]
**Vulnerability:** Internal APIs orchestrating rate-limited or paid external services (e.g. LLMs and external web3 services) were unauthenticated, allowing unauthorized API quota depletion and potential Denial of Service (DoS) attacks. Exception handlers in these API endpoints also returned raw error strings, leading to potential information leakage.
**Learning:** Endpoints like `/check` and `/extract` consume paid APIs and compute, making them vectors for resource exhaustion if exposed without authentication. Exception messages from libraries can contain sensitive internal state.
**Prevention:** Always authenticate internal APIs using API Key headers (e.g., `fastapi.security.APIKeyHeader`) and compare keys using constant-time comparison (e.g., `secrets.compare_digest`). Never return raw exceptions to clients; instead log the exception internally using `logging.error(..., exc_info=True)` and return generic error messages.
